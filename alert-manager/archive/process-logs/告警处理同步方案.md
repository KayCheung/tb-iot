# 告警处理与 ThingsBoard 同步方案

## 需求说明

告警处理过程中，**确认（ACK）**和**清除（CLEAR）**操作需要同步调用 ThingsBoard 接口，保持双向状态一致。

---

## 一、同步操作映射

### 需要同步的操作

| 操作 | ThingsBoard API | 说明 |
|------|----------------|------|
| 确认告警 | `POST /api/alarm/{alarmId}/ack` | 设置 ack_ts，触发 ALARM_ACK 事件 |
| 清除告警 | `POST /api/alarm/{alarmId}/clear` | 设置 clear_ts，触发 ALARM_CLEAR 事件 |

### 不需要同步的操作

| 操作 | 说明 |
|------|------|
| 委托告警 | 本地业务，ThingsBoard 不关心 |
| 处置告警 | 本地业务，ThingsBoard 不关心 |
| 添加处理记录 | 本地业务 |

---

## 二、同步流程设计

### 2.1 确认告警流程

```
用户点击"确认告警"
    ↓
1. 参数校验
    ↓
2. 调用 ThingsBoard API ⚠️ 先同步
   POST /api/alarm/{tb_alarm_id}/ack
    ↓
3. ThingsBoard 返回成功
    ↓
4. 更新本地数据库（事务）
   - acknowledged_time = NOW()
   - acknowledged_by = current_user_id
   - alarm_status = 'ACKNOWLEDGED'
    ↓
5. 记录处理日志和状态历史
    ↓
6. 返回成功
```

### 2.2 清除告警流程

```
用户点击"清除告警"
    ↓
1. 参数校验
    ↓
2. 调用 ThingsBoard API ⚠️ 先同步
   POST /api/alarm/{tb_alarm_id}/clear
    ↓
3. ThingsBoard 返回成功
    ↓
4. 更新本地数据库（事务）
   - cleared_time = NOW()
   - alarm_status = 'CLEARED'
    ↓
5. 记录处理日志和状态历史
    ↓
6. 返回成功
```

## 三、代码实现

### 3.1 告警处理服务

```java
@Service
@Slf4j
public class AlarmHandleService {
    
    @Autowired
    private ThingsBoardClient thingsBoardClient;
    
    @Autowired
    private AlarmInstanceMapper alarmInstanceMapper;
    
    /**
     * 确认告警
     */
    @Transactional(rollbackFor = Exception.class)
    public void acknowledgeAlarm(Long alarmId, Long userId, String remark) {
        // 1. 查询告警
        AlarmInstance alarm = alarmInstanceMapper.selectById(alarmId);
        if (alarm == null) {
            throw new BusinessException("告警不存在");
        }
        
        // 2. 检查是否已确认
        if (isTimeSet(alarm.getAcknowledgedTime())) {
            throw new BusinessException("告警已确认");
        }
        
        // 3. ⚠️ 先调用 ThingsBoard API
        try {
            thingsBoardClient.ackAlarm(alarm.getTbAlarmId());
        } catch (Exception e) {
            log.error("调用 ThingsBoard ACK 接口失败: alarmId={}, tbAlarmId={}", 
                alarmId, alarm.getTbAlarmId(), e);
            throw new BusinessException("确认告警失败：" + e.getMessage());
        }
        
        // 4. 更新本地数据库
        LocalDateTime now = LocalDateTime.now();
        alarm.setAcknowledgedTime(now);
        alarm.setAcknowledgedBy(userId);
        alarm.setAcknowledgedRemark(remark);
        alarm.setAlarmStatus("ACKNOWLEDGED");
        alarmInstanceMapper.updateById(alarm);
        
        // 5. 记录处理日志
        saveHandleLog(alarmId, "ACKNOWLEDGE", "确认告警", remark, userId);
        
        // 6. 记录状态历史
        saveStatusHistory(alarmId, "ACTIVE", "ACKNOWLEDGED", userId, remark);
    }
    
    /**
     * 清除告警
     */
    @Transactional(rollbackFor = Exception.class)
    public void clearAlarm(Long alarmId, Long userId, String remark) {
        // 1. 查询告警
        AlarmInstance alarm = alarmInstanceMapper.selectById(alarmId);
        if (alarm == null) {
            throw new BusinessException("告警不存在");
        }
        
        // 2. 检查是否已清除
        if (isTimeSet(alarm.getClearedTime())) {
            throw new BusinessException("告警已清除");
        }
        
        // 3. ⚠️ 先调用 ThingsBoard API
        try {
            thingsBoardClient.clearAlarm(alarm.getTbAlarmId());
        } catch (Exception e) {
            log.error("调用 ThingsBoard CLEAR 接口失败: alarmId={}, tbAlarmId={}", 
                alarmId, alarm.getTbAlarmId(), e);
            throw new BusinessException("清除告警失败：" + e.getMessage());
        }
        
        // 4. 更新本地数据库
        LocalDateTime now = LocalDateTime.now();
        String fromStatus = alarm.getAlarmStatus();
        alarm.setClearedTime(now);
        alarm.setAlarmStatus("CLEARED");
        alarmInstanceMapper.updateById(alarm);
        
        // 5. 记录处理日志
        saveHandleLog(alarmId, "CLEAR", "清除告警", remark, userId);
        
        // 6. 记录状态历史
        saveStatusHistory(alarmId, fromStatus, "CLEARED", userId, remark);
    }
    
    /**
     * 委托告警（不需要同步到 ThingsBoard）
     */
    @Transactional(rollbackFor = Exception.class)
    public void assignAlarm(Long alarmId, Long assigneeId, String assigneeName, Long userId) {
        // 1. 查询告警
        AlarmInstance alarm = alarmInstanceMapper.selectById(alarmId);
        if (alarm == null) {
            throw new BusinessException("告警不存在");
        }
        
        // 2. 更新本地数据库（不需要调用 ThingsBoard）
        alarm.setAssigneeId(assigneeId);
        alarm.setAssigneeName(assigneeName);
        alarmInstanceMapper.updateById(alarm);
        
        // 3. 记录处理日志
        saveHandleLog(alarmId, "ASSIGN", "委托告警", 
            "委托给：" + assigneeName, userId);
        
        // 4. 发送通知给委托人
        notifyService.notifyAssignee(alarm, assigneeId);
    }
    
    /**
     * 处置告警（不需要同步到 ThingsBoard）
     */
    @Transactional(rollbackFor = Exception.class)
    public void resolveAlarm(Long alarmId, String stepDescription, 
                            String handleResult, List<String> imageUrls, Long userId) {
        // 1. 查询告警
        AlarmInstance alarm = alarmInstanceMapper.selectById(alarmId);
        if (alarm == null) {
            throw new BusinessException("告警不存在");
        }
        
        // 2. 更新本地数据库（不需要调用 ThingsBoard）
        LocalDateTime now = LocalDateTime.now();
        String fromStatus = alarm.getAlarmStatus();
        alarm.setResolvedTime(now);
        alarm.setAlarmStatus("RESOLVED");
        alarmInstanceMapper.updateById(alarm);
        
        // 3. 记录处理日志
        AlarmHandleLog log = new AlarmHandleLog();
        log.setAlarmId(alarmId);
        log.setHandleType("HANDLE");
        log.setStepDescription(stepDescription);
        log.setHandleResult(handleResult);
        log.setImageUrls(JSON.toJSONString(imageUrls));
        log.setOperatorId(userId);
        alarmHandleLogMapper.insert(log);
        
        // 4. 记录状态历史
        saveStatusHistory(alarmId, fromStatus, "RESOLVED", userId, handleResult);
    }
    
    /**
     * 判断时间是否已设置
     */
    private boolean isTimeSet(LocalDateTime time) {
        return time != null && time.isAfter(LocalDateTime.of(1970, 1, 1, 0, 0, 0));
    }
}
```

### 3.2 ThingsBoard 客户端

```java
@Component
@Slf4j
public class ThingsBoardClient {
    
    @Value("${thingsboard.url}")
    private String thingsBoardUrl;
    
    @Value("${thingsboard.username}")
    private String username;
    
    @Value("${thingsboard.password}")
    private String password;
    
    private String token;
    
    private final RestTemplate restTemplate = new RestTemplate();
    
    /**
     * 登录获取 Token
     */
    public void login() {
        String url = thingsBoardUrl + "/api/auth/login";
        
        Map<String, String> request = new HashMap<>();
        request.put("username", username);
        request.put("password", password);
        
        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(
                url, request, Map.class
            );
            
            if (response.getStatusCode().is2xxSuccessful()) {
                Map<String, Object> body = response.getBody();
                this.token = (String) body.get("token");
                log.info("ThingsBoard 登录成功");
            }
        } catch (Exception e) {
            log.error("ThingsBoard 登录失败", e);
            throw new BusinessException("ThingsBoard 登录失败");
        }
    }
    
    /**
     * 确认告警
     */
    public void ackAlarm(String alarmId) {
        ensureToken();
        
        String url = thingsBoardUrl + "/api/alarm/" + alarmId + "/ack";
        
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + token);
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<String> entity = new HttpEntity<>(headers);
        
        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(
                url, entity, Map.class
            );
            
            if (!response.getStatusCode().is2xxSuccessful()) {
                throw new BusinessException("ThingsBoard ACK 接口调用失败");
            }
            
            log.info("ThingsBoard 确认告警成功: alarmId={}", alarmId);
        } catch (Exception e) {
            log.error("ThingsBoard 确认告警失败: alarmId={}", alarmId, e);
            throw new BusinessException("ThingsBoard 确认告警失败：" + e.getMessage());
        }
    }
    
    /**
     * 清除告警
     */
    public void clearAlarm(String alarmId) {
        ensureToken();
        
        String url = thingsBoardUrl + "/api/alarm/" + alarmId + "/clear";
        
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + token);
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<String> entity = new HttpEntity<>(headers);
        
        try {
            ResponseEntity<Map> response = restTemplate.postForEntity(
                url, entity, Map.class
            );
            
            if (!response.getStatusCode().is2xxSuccessful()) {
                throw new BusinessException("ThingsBoard CLEAR 接口调用失败");
            }
            
            log.info("ThingsBoard 清除告警成功: alarmId={}", alarmId);
        } catch (Exception e) {
            log.error("ThingsBoard 清除告警失败: alarmId={}", alarmId, e);
            throw new BusinessException("ThingsBoard 清除告警失败：" + e.getMessage());
        }
    }
    
    /**
     * 确保 Token 有效
     */
    private void ensureToken() {
        if (token == null || token.isEmpty()) {
            login();
        }
    }
}
```

## 五、状态一致性保证

### 5.1 操作顺序

⚠️ **关键原则：先同步 ThingsBoard，再更新本地**

```
正确顺序：
1. 调用 ThingsBoard API
2. API 成功返回
3. 更新本地数据库
4. 提交事务

错误顺序（不要这样做）：
1. 更新本地数据库
2. 调用 ThingsBoard API
3. API 失败
4. 本地已更新，状态不一致 ❌
```

### 5.2 事务管理

```java
@Transactional(rollbackFor = Exception.class)
public void acknowledgeAlarm(...) {
    // 1. ThingsBoard API 调用（在事务外，但失败会抛异常）
    thingsBoardClient.ackAlarm(alarm.getTbAlarmId());
    
    // 2. 本地数据库更新（在事务内）
    alarmInstanceMapper.updateById(alarm);
    
    // 如果 API 调用失败，抛异常，事务回滚
    // 如果 API 成功，本地更新失败，事务回滚
}
```

### 5.3 补偿机制（可选）

如果 ThingsBoard API 成功，但本地更新失败：

```java
try {
    // 调用 ThingsBoard API
    thingsBoardClient.ackAlarm(alarm.getTbAlarmId());
    
    // 更新本地数据库
    alarmInstanceMapper.updateById(alarm);
    
} catch (Exception e) {
    // 记录失败，后续通过定时任务补偿
    saveFailedOperation(alarmId, "ACK", e.getMessage());
    throw e;
}
```

## 七、总结

### 关键点

1. ✅ **确认和清除必须同步**：调用 ThingsBoard API
2. ✅ **委托和处置不需要同步**：本地业务
3. ✅ **先同步后更新**：保证状态一致性
4. ✅ **事务管理**：API 失败时回滚本地操作
5. ✅ **异常处理**：记录失败日志，提供友好提示

### 操作映射表

| 操作 | 同步 ThingsBoard | 更新本地 | 记录日志 | 发送通知 |
|------|-----------------|---------|---------|---------|
| 确认告警 | ✅ ACK API | ✅ | ✅ | ❌ |
| 清除告警 | ✅ CLEAR API | ✅ | ✅ | ❌ |
| 委托告警 | ❌ | ✅ | ✅ | ✅ |
| 处置告警 | ❌ | ✅ | ✅ | ❌ |

---

**实施建议**：优先实现确认和清除的同步逻辑，确保状态一致性。✅
