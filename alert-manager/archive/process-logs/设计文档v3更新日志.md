## 主要变更

### 1. 数据库设计规范更新

#### 非空约束规范
- **所有字段必须定义为 NOT NULL**
- **可空字段必须设置合理的默认值**
- **避免使用 NULL 值，使用默认值代替**

#### 默认值设置规范

**字符串类型**：
```sql
column_name VARCHAR(100) NOT NULL DEFAULT '' COMMENT '描述'
```

**数值类型**：
```sql
column_count INT NOT NULL DEFAULT 0 COMMENT '计数'
column_amount DECIMAL(20,6) NOT NULL DEFAULT 0.000000 COMMENT '金额'
```

**时间类型**：
```sql
-- 未设置时间使用 '1970-01-01 00:00:00'
column_time DATETIME NOT NULL DEFAULT '1970-01-01 00:00:00' COMMENT '时间'

-- 永久有效使用 '9999-12-31 23:59:59'
end_time DATETIME NOT NULL DEFAULT '9999-12-31 23:59:59' COMMENT '结束时间'
```

**JSON 类型**：
```sql
config_json JSON NOT NULL DEFAULT (JSON_OBJECT()) COMMENT '配置JSON'
list_json JSON NOT NULL DEFAULT (JSON_ARRAY()) COMMENT '列表JSON'
```

**外键字段**：
```sql
foreign_id BIGINT NOT NULL DEFAULT 0 COMMENT '外键ID，0表示未关联'
```

## 设计优势

### 1. 性能优势
- 避免 NULL 值判断，索引效率更高
- MySQL 对 NOT NULL 字段有更好的存储优化

### 2. 代码优势
- 不需要频繁进行 NULL 检查
- 减少 NPE（空指针异常）
- 与 MyBatis-Plus 等 ORM 框架配合更好

### 3. 业务优势
- 通过默认值表达业务含义
- 数据一致性更好
- 查询逻辑更简单

## 默认值约定总结

| 类型 | 默认值 | 含义 |
|------|--------|------|
| 时间（未设置） | `'1970-01-01 00:00:00'` | 未设置/未发生 |
| 时间（永久） | `'9999-12-31 23:59:59'` | 永久有效 |
| 外键ID | `0` | 未关联 |
| 计数器 | `0` | 初始值 |
| 状态 | `1` | 启用 |
| 字符串 | `''` | 空字符串 |
| JSON对象 | `(JSON_OBJECT())` | `{}` |
| JSON数组 | `(JSON_ARRAY())` | `[]` |
