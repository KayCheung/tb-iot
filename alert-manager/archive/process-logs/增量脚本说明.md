## 后续维护

### device_count 字段更新时机

1. **创建告警方案**
   - 同步设备到 ThingsBoard 后更新

2. **更新告警方案**
   - 测站关联变化时重新统计

3. **定时同步**
   - 每天凌晨执行同步任务

### 更新 SQL 示例

```sql
-- 更新单个告警方案
UPDATE alarm_scheme 
SET device_count = (
  SELECT COUNT(DISTINCT device_id) 
  FROM alarm_scheme_device_rule 
  WHERE scheme_id = ? AND sync_status = 'SUCCESS'
)
WHERE id = ?;

-- 批量更新所有告警方案
UPDATE alarm_scheme s
SET device_count = (
  SELECT COUNT(DISTINCT device_id) 
  FROM alarm_scheme_device_rule r
  WHERE r.scheme_id = s.id AND r.sync_status = 'SUCCESS'
)
WHERE deleted = 0;
```

## 注意事项

1. 只统计 `sync_status = 'SUCCESS'` 的设备
2. 使用 `DISTINCT` 去重
3. 在事务中更新，保证一致性
4. 大批量操作可以异步更新
