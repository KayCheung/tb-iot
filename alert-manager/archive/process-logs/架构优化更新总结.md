## 三、数据库表结构

### 3.1 是否需要调整？

**结论**：❌ 不需要调整

**原因**：
- `alarm_scheme` 表的 `enable_rule_type` 和 `enable_rule_config` 字段保持不变
- 这些配置用于生成 TBEL 脚本，不影响表结构
- 数据存储方式不变，只是使用方式变化

### 3.2 相关表字段

```sql
-- alarm_scheme 表
enable_rule_type VARCHAR(20) NOT NULL DEFAULT 'ALWAYS'
  -- 值：ALWAYS, SCHEDULED, CUSTOM, CONDITION

enable_rule_config JSONB NOT NULL DEFAULT '{}'
  -- SCHEDULED: {"startDate": "2026-03-01", "endDate": "2026-12-31", ...}
  -- CUSTOM: {"weekDays": [1,2,3,4,5], "timeRanges": [...]}
  -- CONDITION: {"conditionMetric": "pump_status", ...}
```

## 四、接口变更

### 4.1 是否需要调整？

**结论**：❌ 不需要调整

**原因**：
- 前端配置接口不变
- ThingsBoard API 调用不变

### 4.2 相关接口

```
POST /api/alarm-scheme
  - 请求体包含 enableRuleType 和 enableRuleConfig
  - 后端根据配置生成 TBEL 脚本
  - 同步到 ThingsBoard

POST /api/calculatedField (ThingsBoard)
  - expression 字段包含完整的 TBEL 脚本
  - 脚本中包含启动规则判断逻辑
```
