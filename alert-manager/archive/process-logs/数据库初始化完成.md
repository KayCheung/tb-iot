### 告警方案相关（5张表）
1. alarm_scheme - 告警方案表（含 device_count 字段）
2. alarm_scheme_station - 告警方案测站关联表
3. alarm_scheme_metric_rule - 告警方案指标规则表

### 告警记录相关（4张表）

7. alarm_instance_station - 告警实例测站关联表
8. alarm_handle_log - 告警处理记录表
9. alarm_status_history - 告警状态历史表

### 通知管理相关（5张表）
10. notify_rule - 通知规则表
11. notify_template - 通知模板表
12. notify_user_group - 通知用户组表
13. notify_user_group_member - 通知用户组成员表
14. notify_record - 通知记录表

## 执行方式

```bash
# 1. 创建数据库
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS iot_alarm CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. 执行初始化脚本
mysql -u root -p iot_alarm < db/init-schema.sql

# 3. 验证表结构
mysql -u root -p iot_alarm -e "SHOW TABLES;"
```

## 关键设计点

### 1. 非空约束
- 所有字段定义为 NOT NULL
- 可空字段设置合理默认值

### 2. 默认值约定
- 时间未设置：`'1970-01-01 00:00:00'`
- 外键未关联：`0`
- 字符串可选：`''`
- JSON 对象：`(JSON_OBJECT())`
- JSON 数组：`(JSON_ARRAY())`

### 3. 冗余字段
- `alarm_scheme.device_count`：关联设备数量
- 提升查询性能
- 需要在应用层维护一致性
