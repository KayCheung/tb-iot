## 二、核心业务流程设计

### 2.1 告警消息消费与过滤流程

#### 2.1.1 流程说明

```
1. Kafka消费者监听ThingsBoard告警Topic
2. 接收到告警消息后解析消息体
3. 根据设备ID查询设备信息和关联的测站信息
4. 根据测站ID查询关联的告警方案
5. 判断告警方案的启动规则是否满足
6. 判断是否为联动触发，如果是则检查联动条件
7. 如果所有条件满足，创建或更新告警实例
8. 发送告警消息到通知模块的Kafka Topic
```

#### 2.1.2 核心判断逻辑

**启动规则判断**：
- ALWAYS：始终启动，直接通过
- SCHEDULED：检查当前时间是否在指定时间范围内
- CUSTOM：检查当前是星期几，以及时间是否在配置的时间段内
- CONDITION：检查工况条件（预留，后续扩展）

**联动触发判断**：
- 如果触发类型为INDEPENDENT，直接判断单个指标是否满足条件
- 如果触发类型为LINKAGE，需要查询同一linkage_group_id下的所有指标规则
- 从ThingsBoard消息或最近遥测数据中获取所有联动指标的值
- 判断所有联动指标是否同时满足各自的条件
- 只有全部满足时才触发告警

### 2.2 告警方案匹配算法

```java
/**
 * 告警方案匹配流程
 */
public AlarmScheme matchAlarmScheme(Long deviceId, String metricKey) {
    // 1. 查询设备关联的所有测站
    List<Station> stations = stationDeviceRelMapper.selectStationsByDeviceId(deviceId);
    
    // 2. 遍历测站，查询每个测站关联的告警方案
    for (Station station : stations) {
        List<AlarmScheme> schemes = alarmSchemeStationMapper.selectSchemesByStationId(station.getId());
        
        // 3. 遍历告警方案，检查是否包含该指标
        for (AlarmScheme scheme : schemes) {
            List<AlarmSchemeMetricRule> metricRules = 
                alarmSchemeMetricRuleMapper.selectBySchemeIdAndMetricKey(scheme.getId(), metricKey);
            
            if (!metricRules.isEmpty()) {
                // 4. 检查启动规则
                if (checkEnableRule(scheme)) {
                    return scheme;
                }
            }
        }
    }
    
    return null;
}
```

### 2.3 启动规则判断逻辑

```java
/**
 * 检查启动规则是否满足
 */
public boolean checkEnableRule(AlarmScheme scheme) {
    String enableRuleType = scheme.getEnableRuleType();
    
    if ("ALWAYS".equals(enableRuleType)) {
        return true;
    }
    
    JSONObject config = JSONObject.parseObject(scheme.getEnableRuleConfig());
    LocalDateTime now = LocalDateTime.now();
    
    if ("SCHEDULED".equals(enableRuleType)) {
        // 定时启动：检查当前时间是否在指定时间范围内
        String startTime = config.getString("startTime"); // 格式：2026-03-01 08:00:00
        String endTime = config.getString("endTime");     // 格式：2026-03-01 20:00:00
        
        LocalDateTime start = LocalDateTime.parse(startTime, DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        LocalDateTime end = LocalDateTime.parse(endTime, DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss"));
        
        return now.isAfter(start) && now.isBefore(end);
    }
    
    if ("CUSTOM".equals(enableRuleType)) {
        // 自定义时间启动：检查星期几和时间段
        JSONArray weekDays = config.getJSONArray("weekDays"); // [1,2,3,4,5] 表示周一到周五
        JSONArray timeRanges = config.getJSONArray("timeRanges"); // [{"start":"08:00","end":"12:00"},{"start":"14:00","end":"18:00"}]
        
        int currentDayOfWeek = now.getDayOfWeek().getValue(); // 1-7
        if (!weekDays.contains(currentDayOfWeek)) {
            return false;
        }
        
        LocalTime currentTime = now.toLocalTime();
        for (int i = 0; i < timeRanges.size(); i++) {
            JSONObject range = timeRanges.getJSONObject(i);
            LocalTime start = LocalTime.parse(range.getString("start"));
            LocalTime end = LocalTime.parse(range.getString("end"));
            
            if (currentTime.isAfter(start) && currentTime.isBefore(end)) {
                return true;
            }
        }
        
        return false;
    }
    
    if ("CONDITION".equals(enableRuleType)) {
        // 工况启动：预留扩展
        // 可以根据设备状态、其他指标值等条件判断
        return true;
    }
    
    return false;
}
```

### 2.4 联动触发处理逻辑

```java
/**
 * 检查联动触发条件
 */
public boolean checkLinkageTrigger(AlarmScheme scheme, Long deviceId, String metricKey, 
                                   BigDecimal metricValue, JSONObject tbMessage) {
    // 1. 查询该指标的规则
    AlarmSchemeMetricRule currentRule = 
        alarmSchemeMetricRuleMapper.selectBySchemeIdAndMetricKey(scheme.getId(), metricKey);
    
    if (!"LINKAGE".equals(currentRule.getTriggerType())) {
        // 独立触发，直接判断当前指标
        return checkMetricCondition(currentRule, metricValue);
    }
    
    // 2. 联动触发，查询同一联动组的所有规则
    Long linkageGroupId = currentRule.getLinkageGroupId();
    List<AlarmSchemeMetricRule> linkageRules = 
        alarmSchemeMetricRuleMapper.selectByLinkageGroupId(linkageGroupId);
    
    // 3. 获取所有联动指标的当前值
    Map<String, BigDecimal> metricValues = new HashMap<>();
    metricValues.put(metricKey, metricValue);
    
    // 从ThingsBoard消息中获取其他指标值
    JSONObject telemetry = tbMessage.getJSONObject("telemetry");
    for (AlarmSchemeMetricRule rule : linkageRules) {
        if (!rule.getMetricKey().equals(metricKey)) {
            BigDecimal value = telemetry.getBigDecimal(rule.getMetricKey());
            if (value == null) {
                // 如果消息中没有，可以调用ThingsBoard API获取最新遥测数据
                value = getLatestTelemetryFromTB(deviceId, rule.getMetricKey());
            }
            metricValues.put(rule.getMetricKey(), value);
        }
    }
    
    // 4. 判断所有联动指标是否都满足条件
    for (AlarmSchemeMetricRule rule : linkageRules) {
        BigDecimal value = metricValues.get(rule.getMetricKey());
        if (value == null || !checkMetricCondition(rule, value)) {
            return false;
        }
    }
    
    return true;
}

/**
 * 检查单个指标是否满足条件
 */
private boolean checkMetricCondition(AlarmSchemeMetricRule rule, BigDecimal value) {
    // 查询该指标的所有级别条件
    List<AlarmLevelCondition> conditions = 
        alarmLevelConditionMapper.selectByMetricRuleId(rule.getId());
    
    for (AlarmLevelCondition condition : conditions) {
        if (evaluateCondition(condition, value)) {
            return true;
        }
    }
    
    return false;
}

/**
 * 评估条件表达式
 */
private boolean evaluateCondition(AlarmLevelCondition condition, BigDecimal value) {
    String operator = condition.getConditionOperator();
    
    switch (operator) {
        case "GT":
            return value.compareTo(condition.getThresholdValue()) > 0;
        case "GTE":
            return value.compareTo(condition.getThresholdValue()) >= 0;
        case "LT":
            return value.compareTo(condition.getThresholdValue()) < 0;
        case "LTE":
            return value.compareTo(condition.getThresholdValue()) <= 0;
        case "EQ":
            return value.compareTo(condition.getThresholdValue()) == 0;
        case "NEQ":
            return value.compareTo(condition.getThresholdValue()) != 0;
        case "BETWEEN":
            return value.compareTo(condition.getThresholdMin()) >= 0 
                && value.compareTo(condition.getThresholdMax()) <= 0;
        default:
            return false;
    }
}
```

```java
public IPage<AlarmListResponse> queryAlarms(AlarmQueryRequest request) {
    Page<AlarmInstance> page = new Page<>(request.getPage(), request.getSize());
    
    // 使用MyBatis-Plus的QueryWrapper构建查询条件
    LambdaQueryWrapper<AlarmInstance> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(request.getStationId() != null, AlarmInstance::getStationIds, request.getStationId())
           .eq(request.getDeviceId() != null, AlarmInstance::getDeviceId, request.getDeviceId())
           .eq(StringUtils.isNotBlank(request.getLevel()), AlarmInstance::getAlarmLevel, request.getLevel())
           .eq(StringUtils.isNotBlank(request.getStatus()), AlarmInstance::getAlarmStatus, request.getStatus())
           .ge(request.getStartTime() != null, AlarmInstance::getFirstTriggerTime, request.getStartTime())
```
