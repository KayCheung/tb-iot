### 2.1 告警方案列表展示时指标去重

**问题**：告警方案可能配置多个指标规则，列表展示时需要去重显示。

**解决方案**：在应用层实现指标去重逻辑

```java
@Service
public class AlarmSchemeService {
    
    /**
     * 查询告警方案列表（包含去重后的指标）
     */
    public Page<AlarmSchemeVO> listSchemes(AlarmSchemeQuery query) {
        // 1. 查询告警方案列表
        Page<AlarmScheme> schemePage = alarmSchemeMapper.selectPage(
            new Page<>(query.getPage(), query.getSize()),
            new LambdaQueryWrapper<AlarmScheme>()
                .eq(AlarmScheme::getDeleted, 0)
                .orderByDesc(AlarmScheme::getCreatedAt)
        );
        
        // 2. 批量查询指标规则
        List<Long> schemeIds = schemePage.getRecords().stream()
            .map(AlarmScheme::getId)
            .collect(Collectors.toList());
        
        List<AlarmSchemeMetricRule> metricRules = metricRuleMapper.selectList(
            new LambdaQueryWrapper<AlarmSchemeMetricRule>()
                .in(AlarmSchemeMetricRule::getSchemeId, schemeIds)
        );
        
        // 3. 按方案ID分组，去重指标
        Map<Long, List<String>> schemeMetricsMap = metricRules.stream()
            .collect(Collectors.groupingBy(
                AlarmSchemeMetricRule::getSchemeId,
                Collectors.mapping(
                    AlarmSchemeMetricRule::getMetricName,
                    Collectors.collectingAndThen(
                        Collectors.toSet(),
                        set -> new ArrayList<>(set)
                    )
                )
            ));
        
        // 4. 组装VO
        List<AlarmSchemeVO> voList = schemePage.getRecords().stream()
            .map(scheme -> {
                AlarmSchemeVO vo = BeanUtil.copyProperties(scheme, AlarmSchemeVO.class);
                List<String> metrics = schemeMetricsMap.getOrDefault(scheme.getId(), Collections.emptyList());
                vo.setMetricNames(String.join(", ", metrics));
                vo.setMetricCount(metrics.size());
                return vo;
            })
            .collect(Collectors.toList());
        
        return new Page<AlarmSchemeVO>()
            .setRecords(voList)
            .setTotal(schemePage.getTotal())
            .setCurrent(schemePage.getCurrent())
            .setSize(schemePage.getSize());
    }
}
```

### 2.2 告警级别条件表支持 TBEL 脚本

**问题**：当前只支持简单的操作符（>、<、=），无法支持复杂条件。

**解决方案**：增加 `tbel_script` 字段（已在增量脚本中完成）

**数据库变更**：
```sql
ALTER TABLE alarm_level_condition 
ADD COLUMN tbel_script TEXT NOT NULL DEFAULT '';
```

**应用层使用**：
```java
@Service
public class TbelScriptGenerator {
    
    /**
     * 生成级别条件的 TBEL 脚本
     */
    public String generateLevelCondition(AlarmLevelCondition condition) {
        // 1. 如果有自定义 TBEL 脚本，直接使用
        if (StringUtils.isNotBlank(condition.getTbelScript())) {
            return condition.getTbelScript();
        }
        
        // 2. 否则根据操作符生成简单条件
        String metricVar = "value";
        switch (condition.getConditionOperator()) {
            case "GT":
                return String.format("%s > %s", metricVar, condition.getThresholdValue());
            case "GTE":
                return String.format("%s >= %s", metricVar, condition.getThresholdValue());
            case "LT":
                return String.format("%s < %s", metricVar, condition.getThresholdValue());
            case "LTE":
                return String.format("%s <= %s", metricVar, condition.getThresholdValue());
            case "EQ":
                return String.format("%s == %s", metricVar, condition.getThresholdValue());
            case "NEQ":
                return String.format("%s != %s", metricVar, condition.getThresholdValue());
            case "BETWEEN":
                return String.format("%s >= %s && %s <= %s", 
                    metricVar, condition.getThresholdMin(),
                    metricVar, condition.getThresholdMax());
            default:
                throw new IllegalArgumentException("不支持的操作符: " + condition.getConditionOperator());
        }
    }
}
```

### 2.3 告警方案设备规则映射表的规则关联

**问题**：
- `alarm_scheme_device_rule` 表只关联了方案、设备、TB设备ID、TB告警类型、TB计算字段ID
- 如何关联到具体的规则？
- 如何判断规则是否变更，实现增量更新？

**解决方案**：增加 `rule_version` 和 `rule_hash` 字段（已在增量脚本中完成）

**数据库变更**：
```sql
ALTER TABLE alarm_scheme_device_rule 
ADD COLUMN rule_version INT NOT NULL DEFAULT 1,
ADD COLUMN rule_hash VARCHAR(64) NOT NULL DEFAULT '';
```

**应用层实现规则哈希计算**：
```java
@Service
public class AlarmSchemeRuleHashService {
    
    @Autowired
    private AlarmSchemeMapper alarmSchemeMapper;
    
    @Autowired
    private AlarmSchemeMetricRuleMapper metricRuleMapper;
    
    @Autowired
    private AlarmLevelConditionMapper levelConditionMapper;
    
    /**
     * 计算告警方案规则的哈希值
     */
    public String calculateRuleHash(Long schemeId) {
        // 1. 查询告警方案
        AlarmScheme scheme = alarmSchemeMapper.selectById(schemeId);
        if (scheme == null) {
            return "";
        }
        
        // 2. 查询指标规则
        List<AlarmSchemeMetricRule> metricRules = metricRuleMapper.selectList(
            new LambdaQueryWrapper<AlarmSchemeMetricRule>()
                .eq(AlarmSchemeMetricRule::getSchemeId, schemeId)
                .orderByAsc(AlarmSchemeMetricRule::getId)
        );
        
        // 3. 查询级别条件
        List<Long> metricRuleIds = metricRules.stream()
            .map(AlarmSchemeMetricRule::getId)
            .collect(Collectors.toList());
        
        List<AlarmLevelCondition> levelConditions = Collections.emptyList();
        if (!metricRuleIds.isEmpty()) {
            levelConditions = levelConditionMapper.selectList(
                new LambdaQueryWrapper<AlarmLevelCondition>()
                    .in(AlarmLevelCondition::getMetricRuleId, metricRuleIds)
                    .orderByAsc(AlarmLevelCondition::getId)
            );
        }
        
        // 4. 拼接规则内容
        StringBuilder ruleContent = new StringBuilder();
        
        // 方案基本信息
        ruleContent.append(scheme.getSchemeCode()).append("|")
                   .append(scheme.getAlarmMessage()).append("|")
                   .append(scheme.getEnableRuleType()).append("|")
                   .append(JSON.toJSONString(scheme.getEnableRuleConfig())).append("|");
        
        // 指标规则
        for (AlarmSchemeMetricRule rule : metricRules) {
            ruleContent.append(rule.getMetricKey()).append("|")
                       .append(rule.getTriggerType()).append("|")
                       .append(rule.getLinkageGroupId()).append("|")
                       .append(rule.getDeviceId()).append("||");
        }
        
        // 级别条件
        for (AlarmLevelCondition condition : levelConditions) {
            ruleContent.append(condition.getAlarmLevel()).append("|")
                       .append(condition.getConditionOperator()).append("|")
                       .append(condition.getThresholdValue()).append("|")
                       .append(condition.getThresholdMin()).append("|")
                       .append(condition.getThresholdMax()).append("|")
                       .append(condition.getDurationSeconds()).append("|")
                       .append(condition.getTbelScript()).append("||");
        }
        
        // 5. 计算 MD5 哈希
        return DigestUtils.md5Hex(ruleContent.toString());
    }
    
    /**
     * 检查设备规则是否需要同步
     */
    public boolean needSync(Long deviceRuleId) {
        AlarmSchemeDeviceRule deviceRule = deviceRuleMapper.selectById(deviceRuleId);
        if (deviceRule == null) {
            return false;
        }
        
        String currentHash = calculateRuleHash(deviceRule.getSchemeId());
        return !currentHash.equals(deviceRule.getRuleHash());
    }
}
```

**应用层实现增量同步**：
```java
@Service
public class AlarmSchemeSyncService {
    
    @Autowired
    private AlarmSchemeRuleHashService ruleHashService;
    
    @Autowired
    private AlarmSchemeDeviceRuleMapper deviceRuleMapper;
    
    @Autowired
    private ThingsBoardClient thingsBoardClient;
    
    @Autowired
    private TbelScriptGenerator tbelGenerator;
    
    /**
     * 增量同步告警方案到 ThingsBoard
     */
    @Transactional
    public void syncSchemeToThingsBoard(Long schemeId) {
        // 1. 计算当前规则哈希
        String currentHash = ruleHashService.calculateRuleHash(schemeId);
        
        // 2. 查询所有设备规则
        List<AlarmSchemeDeviceRule> deviceRules = deviceRuleMapper.selectList(
            new LambdaQueryWrapper<AlarmSchemeDeviceRule>()
                .eq(AlarmSchemeDeviceRule::getSchemeId, schemeId)
        );
        
        // 3. 只同步哈希值变化的规则
        for (AlarmSchemeDeviceRule rule : deviceRules) {
            if (!currentHash.equals(rule.getRuleHash())) {
                try {
                    // 3.1 生成新的 TBEL 脚本
                    String tbelScript = tbelGenerator.generate(schemeId);
                    
                    // 3.2 更新 ThingsBoard CalculatedField
                    if (StringUtils.isNotBlank(rule.getTbCalculatedFieldId())) {
                        thingsBoardClient.updateCalculatedField(
                            rule.getTbCalculatedFieldId(), 
                            tbelScript
                        );
                    } else {
                        // 首次同步，创建 CalculatedField
                        String fieldId = thingsBoardClient.createCalculatedField(
                            rule.getTbDeviceId(),
                            rule.getTbAlarmType(),
                            tbelScript
                        );
                        rule.setTbCalculatedFieldId(fieldId);
                    }
                    
                    // 3.3 更新本地记录
                    rule.setRuleHash(currentHash);
                    rule.setRuleVersion(rule.getRuleVersion() + 1);
                    rule.setSyncStatus("SUCCESS");
                    rule.setSyncTime(LocalDateTime.now());
                    rule.setSyncError("");
                    deviceRuleMapper.updateById(rule);
                    
                } catch (Exception e) {
                    log.error("同步设备规则失败: deviceRuleId={}", rule.getId(), e);
                    rule.setSyncStatus("FAILED");
                    rule.setSyncError(e.getMessage());
                    deviceRuleMapper.updateById(rule);
                }
            }
        }
    }
    
    /**
     * 标记需要同步的设备规则
     * 在更新告警方案、指标规则、级别条件时调用
     */
    public void markDeviceRulesForSync(Long schemeId) {
        deviceRuleMapper.update(null,
            new LambdaUpdateWrapper<AlarmSchemeDeviceRule>()
                .eq(AlarmSchemeDeviceRule::getSchemeId, schemeId)
                .eq(AlarmSchemeDeviceRule::getSyncStatus, "SUCCESS")
                .set(AlarmSchemeDeviceRule::getSyncStatus, "PENDING")
                .set(AlarmSchemeDeviceRule::getSyncTime, LocalDateTime.of(1970, 1, 1, 0, 0, 0))
                .set(AlarmSchemeDeviceRule::getSyncError, "告警规则已更新，等待重新同步")
        );
    }
}
```

**应用层实现更新时自动标记**：
```java
@Service
public class AlarmSchemeService {
    
    @Autowired
    private AlarmSchemeSyncService syncService;
    
    /**
     * 更新告警方案
     */
    @Transactional
    public void updateAlarmScheme(Long id, AlarmSchemeDTO dto) {
        // 1. 更新告警方案
        AlarmScheme scheme = BeanUtil.copyProperties(dto, AlarmScheme.class);
        scheme.setId(id);
        alarmSchemeMapper.updateById(scheme);
        
        // 2. 更新指标规则
        metricRuleService.updateBySchemeId(id, dto.getMetricRules());
        
        // 3. 更新级别条件
        levelConditionService.updateBySchemeId(id, dto.getLevelConditions());
        
        // 4. 标记需要同步的设备规则
        syncService.markDeviceRulesForSync(id);
        
        // 5. 异步同步到 ThingsBoard
        asyncExecutor.execute(() -> {
            syncService.syncSchemeToThingsBoard(id);
        });
    }
}
```

## 问题 3：接口设计和工时计划

### 3.1 告警方案删除需要同步删除 TB 的高级规则配置

**问题**：软删除告警方案时，需要同步删除 ThingsBoard 的 CalculatedField。

**解决方案**：

```java
@Service
public class AlarmSchemeService {
    
    @Autowired
    private ThingsBoardClient thingsBoardClient;
    
    @Autowired
    private AlarmSchemeDeviceRuleMapper deviceRuleMapper;
    
    /**
     * 删除告警方案
     */
    @Transactional
    public void deleteAlarmScheme(Long id) {
        // 1. 查询告警方案
        AlarmScheme scheme = alarmSchemeMapper.selectById(id);
        if (scheme == null) {
            throw new BusinessException("告警方案不存在");
        }
        
        // 2. 查询所有关联的设备规则
        List<AlarmSchemeDeviceRule> deviceRules = deviceRuleMapper.selectList(
            new LambdaQueryWrapper<AlarmSchemeDeviceRule>()
                .eq(AlarmSchemeDeviceRule::getSchemeId, id)
        );
        
        // 3. 批量删除 ThingsBoard 的 CalculatedField
        for (AlarmSchemeDeviceRule rule : deviceRules) {
            if (StringUtils.isNotBlank(rule.getTbCalculatedFieldId())) {
                try {
                    thingsBoardClient.deleteCalculatedField(rule.getTbCalculatedFieldId());
                } catch (Exception e) {
                    log.error("删除 ThingsBoard CalculatedField 失败: fieldId={}", 
                        rule.getTbCalculatedFieldId(), e);
                }
            }
        }
        
        // 4. 软删除本地记录
        scheme.setDeleted(1);
        alarmSchemeMapper.updateById(scheme);
        
        // 5. 删除设备规则映射记录
        deviceRuleMapper.delete(
            new LambdaQueryWrapper<AlarmSchemeDeviceRule>()
                .eq(AlarmSchemeDeviceRule::getSchemeId, id)
        );
    }
}
```

**工时**：已包含在 Task 3.1（告警方案 CRUD，6h）中
