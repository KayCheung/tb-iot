## 一、告警方案创建流程

### 1.1 流程图

```mermaid
flowchart TD
    Start([用户创建告警方案]) --> Input[前端可视化配置]
    Input --> Config{配置内容}
    
    Config --> Basic[基本信息<br/>方案编码/名称/测站类型]
    Config --> Metric[指标规则<br/>指标/操作符/阈值/持续时间]
    Config --> Enable[启动规则<br/>ALWAYS/SCHEDULED/CUSTOM/CONDITION]
    Config --> Station[关联测站]
    
    Basic --> Submit[提交保存]
    Metric --> Submit
    Enable --> Submit
    Station --> Submit
    
    Submit --> Validate{参数校验}
    Validate -->|失败| Error1[返回错误提示]
    Validate -->|成功| SaveDB[(保存到数据库)]
    
    SaveDB --> SaveScheme[保存告警方案<br/>alarm_scheme]
    SaveScheme --> SaveStation[保存测站关联<br/>alarm_scheme_station]
    SaveStation --> SaveMetric[保存指标规则<br/>alarm_scheme_metric_rule]
    SaveMetric --> SaveCondition[保存级别条件<br/>alarm_level_condition]
    
    SaveCondition --> QueryDevice[查询测站关联的设备<br/>调用外部接口]
    QueryDevice --> LoopDevice{遍历每个设备}
    
    LoopDevice --> GenType[生成 tb_alarm_type<br/>{scheme_code}_{device_id}]
    GenType --> GenTBEL[生成 TBEL 脚本<br/>包含启动规则判断]
    
    GenTBEL --> BuildCF[构建 CalculatedField<br/>expression = TBEL 脚本]
    BuildCF --> CallTB[调用 ThingsBoard API<br/>POST /api/calculatedField]
    
    CallTB --> TBSuccess{API 调用成功?}
    TBSuccess -->|失败| SaveFailed[保存映射记录<br/>sync_status=FAILED]
    TBSuccess -->|成功| SaveSuccess[保存映射记录<br/>sync_status=SUCCESS<br/>tb_calculated_field_id]
    
    SaveFailed --> NextDevice{还有设备?}
    SaveSuccess --> NextDevice
    
    NextDevice -->|是| LoopDevice
    NextDevice -->|否| UpdateCount[更新设备数量<br/>device_count]
    
    UpdateCount --> End([创建完成])
    Error1 --> End
```

### 1.2 时序图

```mermaid
sequenceDiagram
    participant User as 用户
    participant Frontend as 前端界面
    participant Backend as 告警管理系统
    participant DB as 数据库
    participant External as 外部系统
    participant TBELGen as TBEL生成器
    participant TB as ThingsBoard
    
    User->>Frontend: 1. 填写告警方案配置
    Frontend->>Backend: 2. POST /api/alarm-scheme
    
    Backend->>Backend: 3. 参数校验
    Backend->>DB: 4. 保存告警方案
    DB-->>Backend: 5. 返回方案ID
    
    Backend->>DB: 6. 保存测站关联
    Backend->>DB: 7. 保存指标规则
    Backend->>DB: 8. 保存级别条件
    
    Backend->>External: 9. 查询测站关联的设备
    External-->>Backend: 10. 返回设备列表
    
    loop 遍历每个设备
        Backend->>Backend: 11. 生成 tb_alarm_type
        Backend->>TBELGen: 12. 生成 TBEL 脚本
        TBELGen-->>Backend: 13. 返回 TBEL 脚本
        
        Backend->>Backend: 14. 构建 CalculatedField
        Backend->>TB: 15. POST /api/calculatedField
        
        alt API 调用成功
            TB-->>Backend: 16. 返回 CalculatedField ID
            Backend->>DB: 17. 保存映射(SUCCESS)
        else API 调用失败
            TB-->>Backend: 18. 返回错误
            Backend->>DB: 19. 保存映射(FAILED)
        end
    end
    
    Backend->>DB: 20. 更新 device_count
    Backend-->>Frontend: 21. 返回创建结果
    Frontend-->>User: 22. 显示创建成功
```

## 二、告警触发流程

### 2.1 流程图

```mermaid
flowchart TD
    Start([设备上报遥测数据]) --> TB[ThingsBoard 接收数据]
    TB --> CF[CalculatedField 计算]
    
    CF --> Eval{TBEL 脚本评估}
    Eval -->|false| NoAlarm[不触发告警]
    Eval -->|true| EvalDetail[TBEL 脚本执行]
    
    EvalDetail --> CheckEnable{启动规则判断}
    CheckEnable -->|ALWAYS| Pass[通过]
    CheckEnable -->|SCHEDULED| CheckTime1{日期+时间段}
    CheckEnable -->|CUSTOM| CheckTime2{星期+时间段}
    CheckEnable -->|CONDITION| CheckCond{工况条件}
    
    CheckTime1 -->|不满足| NoAlarm
    CheckTime1 -->|满足| Pass
    CheckTime2 -->|不满足| NoAlarm
    CheckTime2 -->|满足| Pass
    CheckCond -->|不满足| NoAlarm
    CheckCond -->|满足| Pass
    
    Pass --> CheckMetric{指标条件判断}
    CheckMetric -->|不满足| NoAlarm
    CheckMetric -->|满足| CheckDuration{持续时间判断}
    
    CheckDuration -->|不满足| NoAlarm
    CheckDuration -->|满足| TriggerAlarm[触发告警]
    
    TriggerAlarm --> RuleChain[规则链处理]
    RuleChain --> MQTT[发送 MQTT 消息]
    
    MQTT --> Consumer[告警系统消费消息]
    Consumer --> Parse[解析消息]
    
    Parse --> ExtractType[提取 tb_alarm_type]
    ExtractType --> QueryMapping{查询映射表<br/>alarm_scheme_device_rule}
    
    QueryMapping -->|未找到| LogWarn[记录警告日志]
    QueryMapping -->|找到| GetScheme[获取告警方案]
    
    GetScheme --> CheckStatus{方案状态检查}
    CheckStatus -->|禁用| Ignore[忽略告警]
    CheckStatus -->|启用| QueryDevice[查询设备信息<br/>调用外部接口]
    
    QueryDevice --> QueryStation[查询关联测站<br/>调用外部接口]
    
    QueryStation --> CheckExist{告警实例是否存在?<br/>tb_alarm_id}
    
    CheckExist -->|存在| UpdateInstance[更新告警实例]
    CheckExist -->|不存在| CreateInstance[创建告警实例]
    
    UpdateInstance --> UpdateCount[触发次数+1<br/>更新最后触发时间]
    CreateInstance --> SaveInstance[保存告警实例<br/>alarm_instance]
    
    UpdateCount --> SaveHistory[记录状态历史]
    SaveInstance --> SaveStationRel[保存测站关联<br/>alarm_instance_station]
    SaveStationRel --> SaveHistory
    
    SaveHistory --> TriggerNotify[触发通知]
    TriggerNotify --> End([处理完成])
    
    LogWarn --> End
    Ignore --> End
    NoAlarm --> End
```

### 2.2 时序图

```mermaid
sequenceDiagram
    participant Device as 设备
    participant TB as ThingsBoard
    participant MQTT as MQTT
    participant Consumer as 告警系统
    participant DB as 数据库
    participant External as 外部系统
    participant Notify as 通知服务
    
    Device->>TB: 1. 上报遥测数据
    TB->>TB: 2. CalculatedField 计算
    TB->>TB: 3. TBEL 脚本评估<br/>（包含启动规则判断）
    
    alt 条件满足（启动规则+指标条件+持续时间）
        TB->>TB: 4. 触发告警
        TB->>MQTT: 5. 发送告警消息<br/>{type: "PUMP_TEMP_001_3001"}
        
        MQTT->>Consumer: 6. 消费消息
        Consumer->>Consumer: 7. 解析消息
        
        Consumer->>DB: 8. 查询映射表<br/>WHERE tb_alarm_type=?
        DB-->>Consumer: 9. 返回方案信息
        
        Consumer->>Consumer: 10. 检查方案状态
        
        Consumer->>External: 11. 查询设备信息
        External-->>Consumer: 12. 返回设备信息
        
        Consumer->>External: 13. 查询关联测站
        External-->>Consumer: 14. 返回测站列表
        
        Consumer->>DB: 15. 检查告警实例是否存在
        
        alt 告警实例已存在
            DB-->>Consumer: 16. 返回已存在
            Consumer->>DB: 17. 更新触发次数和时间
        else 告警实例不存在
            DB-->>Consumer: 18. 返回不存在
            Consumer->>DB: 19. 创建告警实例
            Consumer->>DB: 20. 保存测站关联
        end
        
        Consumer->>DB: 21. 记录状态历史
        Consumer->>Notify: 22. 触发通知
        Notify->>Notify: 23. 匹配通知规则
        Notify->>Notify: 24. 发送通知
    else 条件不满足
        TB->>TB: 不触发告警（启动规则或指标条件不满足）
    end
```

## 三、告警处理流程

### 3.1 确认告警流程图

```mermaid
flowchart TD
    Start([用户点击确认告警]) --> Input[输入确认备注]
    Input --> Submit[提交确认]
    
    Submit --> Validate{参数校验}
    Validate -->|失败| Error[返回错误提示]
    Validate -->|成功| QueryAlarm[查询告警实例]
    
    QueryAlarm --> CheckExist{告警是否存在?}
    CheckExist -->|否| Error
    CheckExist -->|是| CheckAcked{是否已确认?}
    
    CheckAcked -->|是| Error
    CheckAcked -->|否| CallTB[调用 ThingsBoard API<br/>POST /api/alarm/{id}/ack]
    
    CallTB --> TBResult{API 调用结果}
    TBResult -->|失败| Error
    TBResult -->|成功| UpdateLocal[更新本地数据库<br/>事务开始]
    
    UpdateLocal --> SetTime[设置 acknowledged_time]
    SetTime --> SetUser[设置 acknowledged_by]
    SetUser --> SetRemark[设置 acknowledged_remark]
    SetRemark --> SetStatus[设置 alarm_status=ACKNOWLEDGED]
    
    SetStatus --> SaveLog[记录处理日志<br/>handle_type=ACKNOWLEDGE]
    SaveLog --> SaveHistory[记录状态历史<br/>ACTIVE→ACKNOWLEDGED]
    
    SaveHistory --> Commit[提交事务]
    Commit --> Success([确认成功])
    Error --> End([结束])
    Success --> End
```

## 四、通知发送流程

### 4.1 流程图

```mermaid
flowchart TD
    Start([告警事件触发]) --> Event{事件类型}
    
    Event -->|ALARM_CREATED| E1[告警创建]
    Event -->|ALARM_LEVEL_CHANGED| E2[级别变化]
    Event -->|ALARM_ACKNOWLEDGED| E3[已确认]
    Event -->|ALARM_CLEARED| E4[已清除]
    
    E1 --> QueryRules[查询通知规则]
    E2 --> QueryRules
    E3 --> QueryRules
    E4 --> QueryRules
    
    QueryRules --> MatchRules{匹配规则}
    MatchRules -->|无匹配| NoNotify[不发送通知]
    MatchRules -->|有匹配| LoopRules{遍历匹配的规则}
    
    LoopRules --> CheckLevel{检查告警级别过滤}
    CheckLevel -->|不匹配| NextRule
    CheckLevel -->|匹配| GetTemplate[获取通知模板]
    
    GetTemplate --> RenderTemplate[渲染模板<br/>替换变量占位符]
    RenderTemplate --> GetUserGroup[获取用户组成员]
    
    GetUserGroup --> LoopUsers{遍历用户}
    LoopUsers --> CreateRecord[创建通知记录<br/>notify_record]
    
    CreateRecord --> CheckChannel{通知渠道}
    CheckChannel -->|WEB| SendWeb[发送站内信]
    CheckChannel -->|EMAIL| SendEmail[发送邮件]
    CheckChannel -->|SMS| SendSMS[发送短信]
    
    SendWeb --> UpdateStatus1[更新发送状态]
    SendEmail --> UpdateStatus1
    SendSMS --> UpdateStatus1
    
    UpdateStatus1 --> NextUser{还有用户?}
    NextUser -->|是| LoopUsers
    NextUser -->|否| NextRule{还有规则?}
    
    NextRule -->|是| LoopRules
    NextRule -->|否| End([通知完成])
    NoNotify --> End
```

## 六、关键流程说明

### 6.1 告警方案创建关键点

1. **前端可视化配置**：用户无需编写代码
2. **TBEL 脚本生成**：后端自动生成包含启动规则的 JavaScript 脚本
3. **批量同步设备**：为每个关联设备创建 CalculatedField
4. **失败重试机制**：同步失败的记录可定时重试

### 6.2 告警触发关键点

1. **ThingsBoard 计算**：TBEL 脚本在 ThingsBoard 执行
2. **启动规则判断**：在 TBEL 脚本中完成，不满足不触发
3. **MQTT 解耦**：通过消息队列异步处理
4. **方案状态检查**：告警系统侧只检查方案是否启用
5. **幂等性处理**：重复消息只更新触发次数

### 6.3 告警处理关键点

1. **先同步后更新**：确保 ThingsBoard 和本地一致
2. **事务保证**：API 失败时回滚本地操作
3. **操作分类**：确认/清除需同步，委托/处置不需要
4. **日志记录**：完整的操作审计

### 6.4 通知发送关键点

1. **规则匹配**：根据事件类型和告警级别匹配
2. **模板渲染**：支持变量占位符
3. **多渠道发送**：WEB/EMAIL/SMS 独立发送
4. **失败重试**：发送失败自动重试

## 七、性能优化点

### 7.1 告警方案创建
- 批量同步使用线程池
- 限流控制（50 次/秒）
- 失败记录异步重试

### 7.2 告警触发
- MQTT 并发消费
- 外部接口结果缓存
- 数据库批量插入
